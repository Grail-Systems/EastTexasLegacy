# East Texas Legacy Coalition (ETLC)
### *Mapping Transparency. Defending Our Resources. Protecting the Future.*

## 🌲 The Mission
The East Texas Legacy Coalition is a data-driven grassroots movement dedicated to the protection of our regional resources and the enforcement of industrial accountability. We believe that transparency in utility agreements and industrial impact studies is not optional—it is a requirement for the survival of our community.

This repository serves as the **Digital Infrastructure** for the coalition, providing a centralized hub for evidence ingestion, data mapping, and public information dissemination.

---

## 🏛 The Architecture
This project is powered by **Grail Systems** and follows a modular "Hub-and-Spoke" architecture designed for high availability and idiot-proof data ingestion.

### The Three Core Hubs:
1. **Project Lufkin:** A specialized task force mapping local energy infrastructure, utility agreements, and industrial footprint data.
2. **Carrizo-Wilcox Aquifer Defense Grid:** A resource protection initiative focused on monitoring water table integrity and industrial runoff risks.
3. **Press & Public Records Intake:** A secure portal for journalists, whistleblowers, and public officials to submit documentation and evidence.

---

## 🛠 Tech Stack (Under the Hood)
To maintain total control and avoid "SaaS-lockout," this system is built on raw, gritty, and resilient technology:

*   **Frontend:** Lean, mobile-responsive HTML5/CSS3 (The "Front Door").
*   **Infrastructure:** DNS and armored security provided by Cloudflare.
*   **Logic Engine:** Python-based data parsing and Bash-driven automation (The "Engine Room").
*   **Management:** Version control via GitHub, treated as the "Master Command Sheet" for all coalition data.

---

## 📂 Repository Structure
* `/index.html` - The serene Piney Woods entry point and mission hub.
* `/project-lufkin.html` - The intake portal for Lufkin-specific industrial data.
* `/aquifer-defense.html` - Technical data and documentation for the Carrizo-Wilcox defense grid.
* `/assets/` - High-resolution imagery and public-facing documents.
* `/.gitignore` - Security filters to ensure zero leakage of private keys or system trash.
